## Design Context

### Users
STEM students and robotics instructors. Students use the platform for simulating robot behavior and hardware assembly. Instructors use it to monitor progress and manage challenges.

### Brand Personality
**Technical, Industrial, Precise.**
The interface should feel like a high-end diagnostic tool or a 1970s mainframe terminal reimagined with modern precision. It’s unimpressed, fast, and dense.

### Aesthetic Direction
**Industrial Utilitarian.**
- **Theme**: Pure Dark (#050505 base).
- **Colors**: Slate-300 for text, Cyan (#00f5d4) for primary accents/status, slate-800 for borders.
- **Typography**: Monospace (IBM Plex Mono style) for data and headers; sharp Sans-serif for body.
- **Reference**: Industrial control panels, NASA mission control, high-end IDEs.

### Design Principles
1. **The Grid is Truth**: Use a strict 4px/8px modular grid. No organic or "soft" elements.
2. **Monochrome Dominance**: Use color only for status or interaction. 90% of the UI should be shades of black/slate.
3. **Information Density**: Don't be afraid of high density, but use clear hierarchy and 1px borders to separate modules.
4. **Interactive Glow**: Hover and active states should use subtle cyan glows to indicate "powering on".
